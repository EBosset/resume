Configuration du design & du thème

Projet : resume

Ce document explique où modifier les couleurs et le thème de l’application (thème global, panneau latéral gauche, couleur d’accent et color switcher).

1. Thème global clair / sombre

Fichier : src/styles.scss

Principaux sélecteurs et variables :

  • :root { --accent-color: #ff5349; }
    Couleur d’accent par défaut utilisée partout dans l’application. ThemeService met à jour cette valeur dynamiquement quand l’utilisateur choisit une autre couleur dans le color switcher.
  • body.theme-light { ... }
    Couleurs de fond et de texte pour le mode clair (fond clair, texte foncé). Modifie ces valeurs pour changer la base du thème clair.
  • body.theme-dark { ... }
    Couleurs de fond et de texte pour le mode sombre (fond sombre, texte clair).
  • body { --mat-sys-primary: var(--accent-color); }
    Connecte la couleur primaire Angular Material à la variable globale --accent-color.

La classe theme-light ou theme-dark est appliquée sur l’élément <body> par ThemeService (voir ci-dessous).

2. Logique de thème (Signals)

Fichier : src/app/theme.service.ts

  • modeSignal = signal<'light' | 'dark'>('dark') – stocke le mode de thème courant.
  • accentSignal = signal<string>('#FF5349') – stocke la couleur d’accent.
  • Dans le bloc effect(), deux choses se passent :
    • body.classList.toggle('theme-dark' ...) /
      body.classList.toggle('theme-light' ...)
    • body.style.setProperty('--accent-color', accent)

Pour changer les valeurs par défaut (sans passer par le switcher), modifier les valeurs initiales de modeSignal et accentSignal.

3. Panneau latéral gauche (navigation)

Fichier : src/app/app.scss

  • .shell__sidenav { background-color: #000000; ... }
    Couleur de fond principale du panneau gauche en mode sombre.
  • body.theme-light .shell__sidenav { background-color: #f1f5f9; ... }
    Couleur de fond du panneau gauche en mode clair.
  • .shell__sidenav { color: #e5e7eb; } – couleur de texte par défaut dans le panneau (mode sombre).
  • .nav__item { ... color: inherit; } – les liens du menu héritent de la couleur de texte du panneau.
  • .nav__item--active, .nav__item:hover { color: var(--accent-color); }
    L’item de menu actif / survolé utilise la couleur d’accent globale.

Pour changer la couleur du panneau, ajuster les propriétés background-color et color dans les sélecteurs ci-dessus.

4. Composant Color switcher

Fichiers :
src/app/components/color-switcher/color-switcher.component.ts
src/app/components/color-switcher/color-switcher.component.scss

  • Dans le fichier TypeScript, la liste des couleurs disponibles est définie par le tableau colors. Exemple :
    [ '#FF5349', '#FF9800', '#FFC107', ... ]
    Ajoute, supprime ou réordonne des valeurs ici pour changer la palette de couleurs proposée dans le switcher.
  • Un clic sur une couleur appelle setAccent(color), ce qui met à jour le ThemeService et donc la variable --accent-color sur <body>.
  • Dans le fichier SCSS, l’apparence du panneau est contrôlée par des sélecteurs comme .switcher, .color-dot, etc. On peut modifier le fond, l’ombre, les bords arrondis, etc., sans impacter le thème global.

5. Récapitulatif

  • Thème global (clair/sombre + accent) : src/styles.scss
  • État & logique du thème (Signals) : src/app/theme.service.ts
  • Panneau latéral (menu gauche) : src/app/app.scss
  • Color switcher (UI + palette) : src/app/components/color-switcher/...

Curseur personnalisé

Le curseur personnalisé est implémenté comme un composant Angular autonome et utilise la couleur d’accent globale (--accent-color) du thème. Il apparaît uniquement lorsque la souris bouge, puis disparaît après un court délai.

Fichiers à modifier

  • Logique du composant
    src/app/components/custom-cursor/custom-cursor.component.ts
  • Template
    src/app/components/custom-cursor/custom-cursor.component.html
  • Styles (design)
    src/app/components/custom-cursor/custom-cursor.component.scss
  • Masquage global du curseur natif (optionnel)
    src/styles.scss – règles sur html, body, body * pour cacher le curseur natif du navigateur.

Logique du composant

Le composant suit la position de la souris avec des signals et gère la visibilité et l’état « pressé ». On ne modifie ici que le comportement (timing, clic), pas le design. 


// src/app/components/custom-cursor/custom-cursor.component.ts

import { Component, HostListener, signal } from '@angular/core';

@Component({
  selector: 'app-custom-cursor',
  standalone: true,
  templateUrl: './custom-cursor.component.html',
  styleUrl: './custom-cursor.component.scss',
})
export class CustomCursorComponent {
  x = signal(0);
  y = signal(0);
  visible = signal(false);
  pressed = signal(false);
  private hideTimeoutId: number | null = null;

  @HostListener('document:mousemove', ['$event'])
  onMouseMove(event: MouseEvent) {
    this.visible.set(true);
    this.x.set(event.clientX);
    this.y.set(event.clientY);

    if (this.hideTimeoutId !== null) {
      window.clearTimeout(this.hideTimeoutId);
    }

    this.hideTimeoutId = window.setTimeout(() => {
      this.visible.set(false);
      this.hideTimeoutId = null;
    }, 500);
  }

  @HostListener('document:mouseleave')
  onMouseLeave() {
    this.visible.set(false);
    this.pressed.set(false);
  }

  @HostListener('document:mousedown')
  onMouseDown() {
    this.pressed.set(true);
  }

  @HostListener('document:mouseup')
  onMouseUp() {
    this.pressed.set(false);
  }
}

Template

Le template lie la position du curseur et ajoute des classes CSS pour la visibilité (cursor--hidden) et l’état pressé (cursor--pressed). 




<div
  class="cursor"
  [class.cursor--hidden]="!visible()"
  [class.cursor--pressed]="pressed()"
  [style.left.px]="x()"
  [style.top.px]="y()"
></div>

Styles et design

Le design visuel est entièrement défini dans le fichier SCSS. Le curseur utilise la couleur d’accent via var(--accent-color). C’est ici que l’on ajuste la taille, le contour et les transitions. 


// src/app/components/custom-cursor/custom-cursor.component.scss

.cursor {
  position: fixed;
  width: 28px;          // taille normale
  height: 28px;
  border-radius: 50%;
  pointer-events: none;
  transform: translate(-50%, -50%);
  background-color: transparent;
  border: 1.5px solid var(--accent-color);  // contour couleur d'accent
  z-index: 9999;
  opacity: 1;
  transition:
    left 0.12s ease-out,  // déplacement fluide
    top 0.12s ease-out,
    border-color 0.15s ease-out,
    opacity 0.15s ease-out;

  &::after {
    content: '';
    position: absolute;
    top: 50%;
    left: 50%;
    width: 6px;          // point intérieur
    height: 6px;
    border-radius: 50%;
    background-color: var(--accent-color);
    transform: translate(-50%, -50%);
  }
}

.cursor--hidden {
  opacity: 0;           // fondu quand inactif / hors de la fenêtre
}

.cursor--pressed {
  width: 42px;          // cercle plus grand au clic
  height: 42px;
  border-width: 0;
  background-color: var(--accent-color); // rempli avec la couleur d'accent
  opacity: 0.5;         // opacité 50 %

  &::after {
    width: 0;           // on cache le point intérieur en mode pressé
    height: 0;
  }
}

Comment modifier le design

  • Taille : modifier width / height dans .cursor (état normal) et .cursor--pressed (état cliqué).
  • Contour : adapter border: 1.5px solid var(--accent-color) dans .cursor.
  • Point intérieur : ajuster le bloc &::after dans .cursor (taille et couleur).
  • Remplissage / opacité au clic : modifier background-color et opacity dans .cursor--pressed.
  • Fluidité du mouvement : ajuster la durée et l’easing des transitions left/top dans la propriété transition de .cursor.
